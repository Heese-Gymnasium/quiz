### Development

1. Issue nehmen und self-assignen

2. Branch erstellen
    ```bash
    git checkout -b peter/development-doku
    ```

3. Änderungen committen und pushen
    ```bash
    git add .
    git commit -s -m "Beschreibung der Änderungen"
    git push origin peter/development-doku
    ```

4. Pull-Request mit dem eigenen Branch erstellen und Issue (z.B. `#14`) in der Beschreibung erwähnen

5. Review von mind. einer Person anfordern

6. Pull-Request auf GitHub mergen oder:
    ```bash
    git checkout main
    git pull origin main
    git merge peter/development-doku
    ```
    Bei Konflikten lokal lösen, dann committen und pushen
